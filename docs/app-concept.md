# App Concept

## 解決したい課題

日用品や食品を買うとき、同じ商品でも店舗によって価格が違うことがあります。特に車移動が多い地域では、安い店を選んでも移動コストや買い回りの手間が増える場合があります。

OKITOKU は、商品価格、店舗、買い物ルート、チラシ、レシート情報を整理し、日常の買い物前に判断しやすくするアプリを目指しています。

## なぜ沖縄県内の価格比較をテーマにしたか

沖縄では、生活圏の中で複数のスーパーやドラッグストアを使い分ける場面があります。価格だけでなく、移動距離、よく行く店舗、地域ごとの店舗候補も判断材料になります。

そのため、全国向けの大きな価格比較サービスではなく、まずは沖縄県内の買い物に寄せた小さなアプリとして設計しています。

## 想定ユーザー

- 日用品や食品をよく購入する人
- 買い物前に安い店舗を確認したい人
- よく買う商品の価格を記録したい人
- チラシやレシートから価格情報を整理したい人
- 移動コストも含めて買い物先を考えたい人

## 使う場面

### 買い物前

よく買う商品の価格や近隣店舗の候補を見て、どこに行くかを考えます。

### 買い物中

店舗や商品を確認し、過去価格や近隣価格との差を見ます。

### 買い物後

レシートや価格情報を記録し、次回以降の比較に使えるデータを増やします。

## MVP として最初に作る範囲

- ホーム画面
- 利用エリア設定
- 店舗一覧・店舗詳細
- 商品一覧・商品詳細
- 今日の買い物画面
- 商品価格の記録・表示
- ローカルデータで動く価格比較
- Supabase 連携の基本構成

## MVP の次に広げる範囲

- レシート解析
- チラシ取り込み
- 価格履歴の可視化
- 価格変動通知
- 店舗別・カテゴリ別フィルタ
- 実データに近い運用フロー

## このアプリで大事にしていること

- 身近な課題をアプリ仕様に落とし込むこと
- 実装済みと検証中の範囲を分けること
- ローカルだけでも動く構成にすること
- バックエンド連携を後から拡張できる構成にすること
- README と docs で、第三者が目的と構成を追えるようにすること
