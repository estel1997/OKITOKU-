## 詰まった点

- ローカルデータと Supabase データの責務分離
- Supabase 未設定時にもアプリが起動できる構成
- 通知を UI、端末トークン、バックエンド処理に分けて考えること
- OCR や PDF など、プラットフォーム差が出る処理の抽象化

## 改善した点

- ローカル repository と remote repository を分けた

- 実装済み、検証中、今後予定を README に分けて記載した
- docs にアプリの目的、データ設計、ロードマップを分けて整理した

## 学んだこと

- 価格比較アプリでは、商品名、店舗、価格履歴の設計が重要
- Flutter アプリでも、UI より先に domain model を整理すると実装しやすい
- BaaS 連携では、未設定・通信失敗・権限設定を前提に設計する必要がある
- ポートフォリオでは、できていることと検証中のことを分けて書くと信頼されやすい
