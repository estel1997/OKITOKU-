# 実装ロードマップ（優先順位）

`UNIMPLEMENTED.md` の穴と荒削り部分を、**依存の少ない順・ユーザー価値**で並べています。完了した項目は本文と `UNIMPLEMENTED.md` を更新してください。

---

## 意図的に後回し（固定）

以下は**このロードマップの「今すぐ」対象外**です（依頼どおり）。

| 区分 | 内容 |
|------|------|
| 実機・検証 | PC / iOS / Android での網羅的な動作確認（権限・通知・ギャラリー等） |
| PC 向け OCR | Web / Windows / macOS 等での **本番 OCR**（当面 `StubOcrEngine`。Android/iOS の ML Kit は従来どおり） |

---

## フェーズ A — アプリ単体で埋められる荒削り（最優先）

| 順 | テーマ | 内容 |
|----|--------|------|
| A1 | 今日の買い物 | **往復 km の手入力**（GPS なし）。デフォルトは那覇周辺目安の固定値 **（実装済み）** |
| A2 | レシート解析 | **`ReceiptTextParser` のルール追加**（コロン区切り・表記ゆれ）。コーパスが溜まったら店別テンプレへ **（コロン行は実装済み・継続）** |
| A3 | 設定・案内 | **設定画面のセクション整理**、本ロードマップの参照、プッシュ周りの説明の整理 **（実装済み）** |
| A4 | 店舗・周辺価格 UI | 近隣比較カードの文言・商品名解決（ID ではなくカタログ名表示など） **（商品名表示＋文言改善を実装済み）** |
| A5 | チラシ UI | ボトムナビへの「チラシ」導線（6 タブは窮屈ならホーム集約のまま） **（タブ追加なしで導線強化を実装済み）** |

---

## フェーズ B — Supabase・同期（バックエンド寄り）

| 順 | テーマ | 内容 |
|----|--------|------|
| B1 | カタログ | **商品マスタのリモート正と同期ポリシー**（差分・オフライン） |
| B2 | ウォッチ・通知 | **プッシュ ON/OFF とアカウント連携**の設計確定 → 実装 |

| B4 | 価格履歴 | **集計ジョブ / RPC**（サーバ側キャッシュ、重い集計のオフロード） |
